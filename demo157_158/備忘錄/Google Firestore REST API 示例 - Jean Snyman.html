Google Firestore REST API 示例



在這篇文章中，我將記錄一些使用 Firestore REST API 的示例。Googles Firestore 數據庫上的大多數示例將指導您通過 npm 安裝 sdk，這也是我在最新項目中所做的，效果很好。Firestore\Firebase 非常棒，尤其是對於快速原型設計。

當我終於準備好將我的項目部署到網絡上時，我意識到我的包非常大，比如 > 2MB。Firestore 是這個大小的最大貢獻者，所以我開始尋找減小包大小的方法。

不幸的是，我已經以最好的方式使用它了。像這樣的東西。

   import { firebase } from "@firebase/app"; 
    import "@firebase/firestore"; 

    const firebaseApp = firebase.initializeApp({ 
        apiKey: "xxxxxxxxxxxxxxxxxxxxxxxxxxx", 
        authDomain: "your-project.firebaseapp.com", 
        databaseURL: "https://your-project.firebaseio.com", 
        projectId: "your-project", 
        storageBucket: "your-project.appspot.com", 
        messagingSenderId: "xxxxxxxxxxxx" 
    }); 

    export const db = firebaseApp.firestore();

然後我決定嘗試使用 REST API 而不是 sdk。我遇到的唯一問題是很難找到有關使用 REST API 的文檔。（當您閱讀本文時，這可能已經改變）

所以這裡有幾個例子。

獲取集合中的所有文檔

這是最簡單的示例之一，它是對您的收藏的普通 Get 請求。

這是假設您沒有對讀取操作設置授權。如果你這樣做，你將不得不添加一個帶有不記名令牌的授權標頭。

axios.get(`https://firestore.googleapis.com/v1/projects/<PROJECTIDHERE>/databases/(default)/documents/<COLLECTIONNAME>`)
     .then(response => { 
         console.log(response); 
        })
        .catch(error => { 
            console.log(error); 
        });

通過id獲取文檔

這裡我們只是將文檔 id 添加到 URL 的末尾，只返回這個文檔。

axios.get(`https://firestore.googleapis.com/v1/projects/<PROJECTIDHERE>/  databases/(default)/documents/<COLLECTIONNAME>/<DOCID>`)
    .then(response => {
        console.log(response);
    })
    .catch(error => {
        console.log(error);
    });

獲取文檔中滿足特定要求的選定字段，將其限制為 4 個結果並按創建日期排序。

在這個例子中，我們發送一個帶有結構化查詢對象的 post 請求，這將在 Firestore 上運行一個查詢並只返回我們感興趣的數據。

axios.post('https://firestore.googleapis.com/v1/projects/<PROJECTIDHERE>/databases/(default)/documents:runQuery', 
        { structuredQuery: 
            { from: [
                { collectionId: 'posts' 
            }
        ], 
        orderBy: [
            { field: 
                { fieldPath: 'created' 
            }, direction: 'DESCENDING' }
        ], select: { fields: 
            [
                { fieldPath: 'title' }, 
                { fieldPath: 'summary' }, 
                { fieldPath: 'category' }, 
                { fieldPath: 'created' }
            ] 
        }, 
        where: { 
            compositeFilter: { 
                filters: [
                    { fieldFilter: { 
                        field: { 
                            fieldPath: 'published' 
                        }, 
                            op: 'EQUAL', 
                            value: { 
                                booleanValue: true 
                            } 
                        } 
                    }
                ], op: 'AND' 
            } 
        }, 
            limit: 4 
            } 
        }
        ).then(res => { 
            console.log(res) 
        })
        .catch(error => { 
            console.log(error) 
        })

創建文檔（發布請求）

在這裡，您只需要注意您發送到 Firestore 的對象的格式。

axios.post("https://firestore.googleapis.com/v1/projects/<PROJECTIDHERE>/databases/(default)/documents/<COLLECTIONNAME>", 
        { 
            fields: { 
                title: { stringValue: this.title }, 
                category: { stringValue: this.category }, 
                post: { stringValue: this.post }, 
                summary: { stringValue: this.description }, 
                published: { booleanValue: this.published }, 
                created: { timestampValue: new Date() }, 
                modified: { timestampValue: new Date() } 
            } 
        }
        ).then(res => { 
            console.log("Post created") 
        })

僅修補特定字段

這是最難弄清楚的部分，經過大量谷歌搜索後，有人引導我朝著正確的方向前進，我終於做對了。除了以與 Post 請求相同的方式傳遞對象之外，您還可以在 URL 查詢參數中指定要更新的字段。

axios.patch(`https://firestore.googleapis.com/v1/projects/<PROJECTIDHERE>/databases/(default)/documents/<COLLECTIONNAME>/<DOCID>?updateMask.fieldPaths=title&updateMask.fieldPaths=post&updateMask.fieldPaths=summary&updateMask.fieldPaths=category &updateMask.fieldPaths=published&updateMask.fieldPaths=modified`, 
        { fields: 
            { title: 
                { stringValue: this.post.title }, 
                post: { stringValue: this.post.post }, 
                summary: { stringValue: this.post.description }, 
                category: { stringValue: this.post.category }, 
                published: { booleanValue: this.post.published }, 
                modified: { timestampValue: new Date() } 
            } 
        }
    )
    .then(doc => { 
        console.log("Document successfully updated!"); 
    })

刪除

這很簡單，傳入您的項目 ID、集合名稱和文檔 ID。

axios.delete(`https://firestore.googleapis.com/v1/projects/<PROJECTIDHERE>/databases/(default)/documents/<COLLECTIONNAME>/<DOCIDHERE>`)
    .then(doc => {
        console.log("Document successfully deleted!"); 
    })

結論

我希望這對也想使用這些 API 的其他人有所幫助。使用 REST 而不是 sdk 對我來說效果很好，我的包大小大大減少了。
